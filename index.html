Setup Instructions

Step 1 — Create a GitHub Personal Access Token (one-time)

  1. 1 Go to GitHub → Settings → Developer settings → Fine-grained tokens
  2. 2 Click Generate new token
  3. 3 Name: COps Weather Dashboard, Expiration: 90 days (or custom)
  4. 4 Repository access: Only select repositories → choose COps-work-automatization
  5. 5 Permissions → Repository: Actions → Read and write, Contents → Read
  6. 6 Click Generate token → copy and paste it in the Token field in the header above

Step 2 — Set up Bolt Admin cookies (one-time, done by Taras)

  1. 1 Log in to admin-panel.bolt.eu
  2. 2 F12 → Network → click any request → right-click → Copy as cURL
  3. 3 Run locally: python automation/parse_curl.py → paste cURL → get base64 string
  4. 4 Go to GitHub repo → Settings → Secrets → Actions
  5. 5 New repository secret: Name = BOLT_COOKIES, Value = the base64 string

How it works

  • Select your name and paste your GitHub token in the header
  • Click a weather button (Good / Bad / Harsh) on any city card
  • This triggers a GitHub Actions workflow that runs Playwright (headless Chrome) on GitHub servers
  • Playwright opens the Bolt Admin panel → merges JSON settings → clicks Update
  • Status updates automatically when the workflow completes (~40-60 sec)
  • No extensions, no servers — everything runs on GitHub for free
